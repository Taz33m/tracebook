# Coinbase Exchange L3 Adapter

`tracebook-coinbase` reconstructs a Coinbase Exchange order book from a REST
level-3 snapshot and a recorded `full` or compact `level3` WebSocket stream. It
normalizes the venue lifecycle into `MarketEvent` records and replays those
records through Tracebook's validated FIFO book.

This is an offline importer. It does not open a WebSocket, handle credentials,
place orders, or claim to be production trading infrastructure.
The included BTC-USD fixture is synthetic and documentation-shaped, not a
redistributed market-data capture. Tracebook is not affiliated with Coinbase.

## Why Snapshot Plus Feed

Coinbase documents a synchronization sequence: subscribe, queue messages, fetch
a REST L3 snapshot, discard queued messages at or before the snapshot sequence,
then apply the remaining messages in order. The adapter enforces that model and
requires every later sequence number to be exactly one greater than the prior
one.

- [Coinbase Exchange WebSocket channels](https://docs.cdp.coinbase.com/exchange/websocket-feed/channels)
- [Coinbase Exchange sequence numbers](https://docs.cdp.coinbase.com/exchange/websocket-feed/overview#sequence-numbers)
- [Coinbase Exchange product book](https://docs.cdp.coinbase.com/api-reference/exchange-api/rest-api/products/get-product-book)

## Command

```bash
tracebook-coinbase \
  examples/data/coinbase_btcusd_l3_snapshot.json \
  examples/data/coinbase_btcusd_full.jsonl \
  --tick-size 0.01 \
  --events-output /tmp/coinbase-events.jsonl \
  --include-trades \
  --output /tmp/coinbase-replay.json
```

`--tick-size` is required. Use the product's Coinbase `quote_increment`; the
adapter will not guess it from observed prices and rejects prices that are not
exact multiples. Use `--product-id` when a saved snapshot does not include that
field. The command also rejects any normalized lifecycle that generates a
simulated match, because synchronized Coinbase `open` events must not cross the
resting book.

The command streams normalized events through the replay engine. When
`--events-output` is present, it writes to a temporary file and atomically moves
the completed JSONL into place, so a failed strict import does not leave a
partial dataset at the requested path.

Options:

| Option | Meaning |
| --- | --- |
| `--tick-size` | Required Coinbase product `quote_increment` |
| `--product-id` | Product identifier when absent from the snapshot |
| `--lenient` | Retain issues and continue with `sequence_complete=false` |
| `--depth-levels` | Number of replay depth levels in the summary |
| `--include-trades` | Retain and include observed Coinbase matches; increases memory use |
| `--include-id-map` | Retain and emit normalized-to-venue order IDs; increases memory use |
| `--events-output` | Optional normalized `MarketEvent` JSONL path |
| `--output` | Optional combined adapter and replay JSON summary path |

Strict mode is the default. It fails on sequence gaps, malformed messages,
unknown maker orders, inconsistent sizes/prices/sides, compact arrays received
before their schema, crossed snapshots, and auction-mode snapshots.

## Lifecycle Mapping

| Coinbase input | Normalized behavior |
| --- | --- |
| REST L3 bid/ask row | `new` resting limit order |
| `received` | Sequence is checked; no book mutation |
| `open` | `new` resting limit order |
| `match` | `reduce` maker remaining size without losing queue priority; preserve an observed exchange trade |
| `done` for an active canceled order | `cancel` |
| `done` for an order never opened or already filled | Ignore, as Coinbase requires for book maintenance |
| Same-price size decrease in `change` | `reduce` without losing priority |
| Price move or size increase in `change` | `replace` (cancel-and-new priority) |
| `change` for an order not on the book | Ignore |
| compact `noop` | Sequence is checked; no book mutation |
| `activate` | Ignore; a triggered stop subsequently follows the normal lifecycle |

Observed Coinbase matches are not inserted into Tracebook's simulated trade
tape. The adapter report calls them `exchange_trades`; replay-generated matches
remain under the separate replay artifact. This prevents source facts from
being confused with executions generated by the simulator.

## Compact Level3

The compact channel sends one schema object followed by arrays. The adapter does
not hard-code array positions: it validates the announced schemas for `open`,
`done`, `match`, `change`, and `noop`, then maps each array through those field
names. An array before the schema is rejected.

## Python API

```python
from tracebook.events import (
    CoinbaseL3Adapter,
    iter_coinbase_l3_messages,
    load_coinbase_l3_snapshot,
)
from tracebook import replay_market_events

snapshot = load_coinbase_l3_snapshot("snapshot.json")
adapter = CoinbaseL3Adapter(snapshot, product_id="BTC-USD", retain_trades=True)

result = replay_market_events(
    adapter.iter_events(iter_coinbase_l3_messages("full.jsonl")),
    matching_algorithm="fifo",
    tick_size=0.01,
)

print(adapter.to_dict(include_trades=True))
print(result.manager.get_order_book("BTC-USD").get_order_book_depth())
```

`CoinbaseL3Adapter.iter_events()` is intentionally single-use and streaming.
The convenience `normalize_coinbase_l3()` API collects the resulting events
when a small fixture or in-memory workflow is more convenient.

## Identifier Mapping

Coinbase order IDs are strings while normalized `MarketEvent.order_id` values
are positive integers. `coinbase_order_id(product_id, venue_order_id)` derives a
deterministic positive 63-bit ID with BLAKE2b. Active collisions are detected
and rejected. `--include-id-map` retains the full mapping for audit output.
Without ID or trade retention, adapter state memory tracks the active book;
lenient-mode issue records grow only with the problems encountered.

## Boundaries

- Snapshot rows must be true L3 `[price, size, order_id]` rows. Aggregated L2
  rows are rejected because they cannot recover individual queue position.
- Snapshot array order is preserved. The adapter does not invent stronger queue
  ordering guarantees than the captured source provides.
- Auction books can be crossed by design and are outside the current matching
  model, so `auction_mode=true` and crossed/locked snapshots are rejected.
- Public feed data does not expose participant ownership. Imported orders use
  Tracebook's anonymous owner and do not reconstruct venue self-trade policy.
- Price and size strings are validated with `Decimal`, then converted to the
  simulator's float-based core representation.
- Lenient mode is for forensics. Any gap or invalid message marks the result
  incomplete; it must not be presented as an exact book reconstruction.

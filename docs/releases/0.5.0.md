# Tracebook 0.5.0: Qualification As A Public Contract

Version 0.5.0 publishes the qualification and external-adoption work that
landed after 0.4.1. It does not change Tracebook into a production exchange or
add a speculative generator. It makes the current matching-engine correctness
workflow available from the public package under a new, honest version.

## Why This Release Exists

Current `main` documented `tracebook-conformance qualify` while still reporting
0.4.1, but the 0.4.1 package on PyPI predated that command. A clean external
engine checkout therefore could not run the documented qualification workflow.
0.5.0 restores the release invariant: the pinned public package contains the
CLI and artifact contracts shown in the README and CI examples.

## Public Qualification

```bash
python -m pip install "tracebook-sim==0.5.0"

tracebook-conformance qualify \
  --profile fifo-limit-v1 \
  --seed 42 \
  --traces 25 \
  --events-per-trace 200 \
  --candidate-cmd './matching-engine --tracebook-stdio' \
  --output-dir .tracebook/qualification
```

The atomic output combines the selected fixed cases, deterministic campaign,
semantic coverage, JSON, JUnit, and any minimized first divergence. A passing
bundle is bounded evidence for the named profile and workload. It is not production certification.

## Included Evidence And Integrations

- Qualification contract version 1 and the public engine-qualification intake.
- A commit-pinned `gocronx/matcher` adapter with a passing `fifo-limit-v1`
  qualification.
- The shared Rust protocol crate used by maintained native adapters.
- `tracebook-conformance-v2`, including the FIFO `CANCEL_RESTING` regression.
- `fifo-partial-fill-v1`, which isolates partial-fill priority retention.
- The Flash-to-Tracebook handoff for the historical `orderbook-rs` issue #88,
  including a one-minimal four-event regression.
- The maintained `orderbook-rs` 0.12.0 and `pricelevel` 0.9.1 integration.
- The frozen guided-discovery experiment, which remains unshipped because it
  did not improve held-out time-to-divergence consistently.

## Compatibility

- Distribution: `tracebook-sim==0.5.0`
- Python: 3.10 or newer
- Conformance protocol: version 1, unchanged
- Artifact schema: version 1, unchanged
- Existing campaign and reproduction commands remain available.
- The default bundled fixed suite is now immutable suite v2; suite v1 remains
  selectable explicitly.

Profiles remain scoped contracts. `fifo-limit-v1` still excludes market, IOC,
FOK, self-trade prevention, pro-rata matching, and in-place quantity increases.

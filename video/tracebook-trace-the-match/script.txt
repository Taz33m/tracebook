Most order book demos lead with speed. tracebook leads with evidence.

It is a Python order-book workbench for matching, event simulation, profiling, and reproducible benchmarks.

The core validates orders, crosses the spread, applies FIFO priority, and can split pro-rata fills by resting size.

Simulation events include new, cancel, and replace, all aimed at active resting order IDs.

Benchmarks separate generation time from book-processing latency, so the numbers stay inspectable.

Reports include seed, warmup, machine metadata, dependencies, percentiles, memory, and overhead. Local baselines stay local.

A local dashboard turns the same simulation into live depth and telemetry.

The repo gives reviewers proof: tests, smoke commands, package builds, CI, and bounded claims.

tracebook: inspect the book, benchmark the path, trace the system.

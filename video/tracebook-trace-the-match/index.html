github.com/Taz33m/tracebook
Most order book demos start with speed. tracebook starts with evidence.
A latency-focused Python order book simulator for matching, synthetic flow, lifecycle events, and profiling.
Orders are validated, matched, filled, partially rested, or cancelled according to explicit semantics.
The simulation path interleaves cancellations and replacements against active resting IDs.
Benchmark reports separate generation time from book-processing latency.
Each report carries seed, warmup, machine metadata, dependencies, and latency percentiles.
A local dashboard turns the same simulation into live depth and telemetry.
The repo is built around tests, smoke commands, package builds, CI, and bounded claims.
